# Collective Branch Overview — 2026-04-13

## What This Is

The `collective` branch is the integration target for Fleet, a browser-based autonomous agent orchestrator. It merges:

- **main** — updated by Gemini Studio directly (UI, modules, orchestrator logic)

## What's New from feat/wasm-executor

### YuanFS (`wasm/boot/yuanfs.go`, 207 lines)
Agent-as-filesystem bridge exposing YUAN agent as virtual files:
- `/#yuan/in` — write-only, accumulates message
- `/#yuan/out` — read triggers `agent.run(msg)`, returns response
- `/#yuan/status` — returns idle/running/error/not initialized
- Requires `window.boardVM.yuan` with `init()`, `send(msg)`, `status()` methods

### idbfs (`wasm/boot/idbfs/idbfs.go`, 1055 lines)
Full IndexedDB-backed filesystem in Go/WASM:
- Persistent overlay for cowfs — environment survives page reloads
- Implements: Open, Stat, ReadDir, OpenFile, Mkdir, MkdirAll, Remove, RemoveAll, Rename, Truncate, Chtimes, Chmod
- Fixed deadlock in openDir→ReadDir flow
- Fixed ModeDir flag handling
- Immediate persist on create for p9 Create→Stat flow
- Uses channel-based awaitIDB helper (avoids Promise wrapping)

### Updated Boot Sequence (`wasm/boot/main.go`, 403 lines)
- Two modes: `executor` (default) and `terminal`
- Wanix kernel + 9p setup via virtio9p
- Environment: `cowfs(rootfs from bundle + idbfs persistent overlay)` → `#env`
- GitFS → `#repo` (read-only, both modes)
- BoardFS → `#board` (terminal mode only)
- LLMFS → `#llm`, ToolFS → `#tools` (both modes)
- YuanFS → `#yuan` (optional, if configured)
- WISP networking: `wisp://host/wisp` for full TCP tunneling
- Control file with commands: cmd, bind, reload, bundle, cp, sync

### Build Pipeline (`wasm/system/repack.sh`)
- Docker build → extract rootfs → overlay custom bins + agent.wasm → tar.gz
- `--docker` flag for full rebuild, otherwise overlay-only repack

### Job Control
- Shell bound to ttyS0 with job control enabled (Ctrl+C support)
